## 릴레이 코딩 테스트
> 알고리즘 스터디에서 대회를 진행하며 없어서 직접 만든 플랫폼

## 기능
    릴레이 코딩 — 참가자들이 랜덤 순서로 코드를 이어서 작성
    실시간 동기화 — Socket.io 기반으로 현재 코드가 모든 참가자에게 실시간 반영
    코드 실행 — Python / C++ / C 코드를 직접 실행하고 결과 확인
    자동 채점 — 턴 넘길 때 테스트케이스 자동 채점
    기여도 분석 — AST 구조 분석으로 참가자별 코드 기여도 계산
    타이머 — 턴당 제한 시간, 초과 시 자동으로 다음 사람으로 넘어감
    보안 샌드박스 — Docker 컨테이너에서 코드 실행, 위험한 명령어 차단

## 기술 스택
Frontend        HTML / CSS / JavaScript
Backend         Node.js / Express
실시간 통신      Socket.io
코드 실행        Docker
코드 분석        AST (언어별 파서)

## 개발 과정에서 고민했던 것들
1. 보안 — 사용자 코드를 어떻게 안전하게 실행할까
사용자가 입력한 코드를 서버에서 직접 실행하면 os, system 같은 위험한 명령어로 서버를 공격할 수 있었다. 
Docker 컨테이너를 샌드박스로 활용해서 네트워크 차단, 메모리 제한, 위험 키워드 블랙리스트를 적용해 해결했다.

2. AST 기반 기여도 계산 — 처음 접하는 개념이라 어려웠다
단순히 줄 수로 기여도를 계산하면 의미없는 코드를 잔뜩 작성해도 점수를 받는 문제가 있었다. 
ex) void  a(){}, void b(){}
AST(추상 구문 트리)로 코드 구조를 분석하는 방식을 처음 접했는데, 언어마다 같은 기능이라도 노드 이름이 달라서 (C++의 class_specifier, Python의 class_definition) 
언어별로 따로 처리해야 했다.

3. 점수 체계 — 공정한 기준이 뭘까
처음 설계한 점수 체계는 의미없는 함수를 만들어도 점수를 받는 허점이 있었다. 
고민 끝에 코드 구조 점수 외에 "내가 작성한 코드가 최종 코드에 얼마나 살아있냐" 를 기준으로 한 생존 점수를 추가했다. 
또한 앞 순서와 뒷 순서의 난이도 차이를 반영한 가중치도 적용했다.

4. 실사용 피드백 반영
스터디원들에게 실제로 사용해보게 하고 피드백을 받았다.
타이머 가시성, 총 인원 표시 등을 개선했고,
추가 요청사항(멀티방, 실시간 코딩 공유, Monaco Editor 등)은 개선 방향으로 기록해두었다.


## 실행 방법
    1. 의존성 설치
        npm install

    2. Docker 설치 필요 (코드 실행 환경)
        # https://www.docker.com/

    3. 서버 실행
        node relaycoding.js

    4. 브라우저에서 접속
        http://localhost:3000
